# E156 Micro-Paper Specification v0.2

## Purpose

`E156` is a compact evidence-synthesis publication format built around one fixed narrative body and one richer external evidence layer. The body is optimized for readability, screenshot survival, and machine validation. The companion HTML carries the data, plots, methods, links, and validation artifacts.

## Hard Body Rules

- Exactly `7` sentences
- Exactly `156` words in release mode
- Single paragraph
- No headings inside the body
- No citations or references inside the body
- No links inside the body
- No metadata inside the body
- One named primary estimand only

## Sentence Contract

1. `Question`
   State the population, intervention or exposure or comparator, and one main endpoint.
2. `Dataset`
   State studies, participants, scope, timing, or follow-up.
3. `Method`
   State synthesis design, model, and effect scale.
4. `Primary result`
   Report the lead estimate with interval.
5. `Robustness`
   Report one heterogeneity, inconsistency, sensitivity, or secondary directional finding.
6. `Interpretation`
   State restrained meaning in plain language.
7. `Boundary`
   State the main limitation, scope limit, or main harm.

## Recommended Internal Word Budget

Use this as a drafting target inside the fixed 156-word total:

- Sentence 1: `22`
- Sentence 2: `20`
- Sentence 3: `20`
- Sentence 4: `30`
- Sentence 5: `22`
- Sentence 6: `22`
- Sentence 7: `20`

## Outside Note Block

Required fields:

- `Type`
- `Primary estimand`
- `App`
- `Data`
- `Code`
- `DOI`
- `Version`
- `Date`
- `Certainty`

Optional fields:

- `Validation`
- `Protocol`
- `Funding`
- `Conflicts`
- `Source article`

## Variant Types

- `pairwise`
- `network`
- `diagnostic`
- `prevalence`
- `methods`

## House Style

- One idea per sentence
- Numbers over adjectives
- No hype
- No causal overreach unless design justifies it
- Limitation is mandatory
- The body must make sense when seen alone

## Publication Model

The preferred publication unit is:

1. `E156 body`
2. `Outside note block`
3. `Interactive HTML companion`

The companion HTML should carry the full evidence layer:

- structured metadata
- tables
- plots
- source links
- code and data links
- validation status
- version history
